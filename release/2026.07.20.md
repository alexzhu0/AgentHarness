# AgentHarness v0.2.0-alpha.1

This is an alpha GitHub source prerelease of the AgentHarness pre-execution
evidence control-plane.

It does not publish a PyPI or npm package. It does not include the Pi companion
implementation.

## Highlights

- Adds the versioned Pi observation and evidence-response contract.
- Adds deterministic AgentHarness-side evidence evaluation and verification.
- Adds the AgentHarness evidence contract for a finite methodology-permit
  pilot, without reading the pinned artifact or executing a tool in this repo.
- Preserves `result_status: not_executed` for pre-execution evidence.
- Removes the superseded `agentharness_proto/` implementation.
- Replaces machine-specific documentation paths and personal test paths with
  portable examples.
- Keeps disclosure tests while constructing the AWS-like canary at runtime
  instead of storing a credential-shaped literal in the repository.

## Runtime and trust boundary

The Pi/Win9 extension used by the pilot is a trusted, version-pinned,
admission-controlled part of the external runtime TCB.

AgentHarness emits `permit_once` only as a deterministic evidence decision. It
has no in-repo execution effect and is not direct runtime authority. The
AgentHarness contract binds the exact call ID, tool name, canonical argument
digest, policy, and pinned artifact digest while retaining
`result_status: not_executed`.

The accepted cross-repository pilot separately demonstrated Pi-side
registration and session binding, single-use consumption, replay and
replacement rejection, and one pinned local read-only action. Those runtime
controls belong to the excluded Pi companion, not this AgentHarness source
release, and do not establish general runtime authorization.

AgentHarness does not provide an extension sandbox. This pilot does not prevent
Pi extensions from using process or network capabilities that Pi already
provides. AgentHarness is not an auth gateway, signer, trust root, governance
authority, or general execution plane.

The Pi companion code remains in the Pi repository and is intentionally
excluded from this AgentHarness release.

## Known limitations

- This is an alpha prerelease, not production-readiness certification.
- It does not authorize arbitrary Pi or Win9 tool execution.
- It does not provide extension process or network containment.
- It does not add signing, timestamping, attestation, sandbox, identity,
  credential, or trust-root infrastructure.
- External runtime owners remain responsible for authorization and execution.

## Upgrade note

Use the repository CLI from this source tree or an explicitly built local
artifact. No registry package is published for this prerelease.
