## Global invariant

AgentHarness is a policy-driven, auditable **pre-execution evidence control-plane**.

It is not a direct runtime. It must not silently become a daemon, scheduler, watcher,
tool executor, auth gateway, sandbox, signing authority, timestamp authority, trust
root, or governance engine.

Current Pi-facing work is still dry-run / contract wiring. `allow_candidate` is evidence
only, not runtime approval.

## Review gates

Before accepting any task:

1. Inspect actual repo status and diff.
2. Confirm changed files match the task scope.
3. Run targeted tests for changed behavior.
4. Run build/typecheck/lint/static probes appropriate to the repo.
5. Verify no forbidden dependency, runtime, or boundary drift appeared.
6. Verify outputs remain deterministic and non-executing where required.
7. Record any watch items separately from blockers.

## Pause rules

Pause instead of continuing if:

- the executor reports success but the diff includes out-of-scope files;
- verification passes but the scope boundary is unclear;
- a task would require production auth, sandbox, signing, timestamping, trust root, or
  real runtime approval;
- a task would convert `allow_candidate` into runtime allow;
- dirty files from unrelated work are present and would be touched;
- implementation needs a new dependency or lockfile change not planned in the task;
- more coding would hide an architectural decision that needs explicit review.

## Source-location contract

Release files live in AgentHarness, but most planning evidence remains in the
CL4R1T4S planning repository. A future standalone AgentHarness release should either
preserve that planning repository or copy immutable plan-artifact hashes into this
release directory.

If used outside this checkout, reviewers should treat CL4R1T4S / Pi evidence as
externally sourced unless a commit hash, review ID, copied artifact, or digest is
recorded here.

## Review reset anchor

Reset review to this date file when future work becomes an unbounded coding loop. The
reset baseline is the dated release index, the review gates, the pause rules, and the
no-go line below.
